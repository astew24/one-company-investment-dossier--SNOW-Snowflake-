I built this as a single-name coverage exercise: variant view, three-case framing, KPI monitoring, and source logging. The point is not a flashy dashboard. It is to show how the thesis would be defended, updated, and pressure-tested after each print.

Everything on this page points back to the memo, tracker, model inputs, and source log in the repo. Illustrative values are labeled, and the workflow stays filing-first.
